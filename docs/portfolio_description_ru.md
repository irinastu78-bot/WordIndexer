# WordKeywords / WordIndexer

WordKeywords, или WordIndexer, — это инструмент для автоматической сборки оглавления и указателей в большом Word-сборнике научных статей. Он берёт один `.docx` с набором статей и создаёт итоговый `.docx`, в который добавлены русские и английские предметные указатели, русские и английские авторские указатели, а также русское оглавление.

Проект решает практическую задачу редакторской подготовки сложного научного сборника: вместо ручного поиска страниц, авторов, ключевых слов и переноса блоков в итоговый документ pipeline извлекает данные из Word, сохраняет проверяемые debug-артефакты и затем вставляет готовые разделы обратно в документ.

## Вход и выход

Вход:

- `.docx` с набором статей;
- статьи должны быть близки к поддерживаемому шаблону;
- в статьях должны быть русские и английские заголовки, авторы и keyword-блоки.

Выход:

- `.docx` с RU keyword index;
- `.docx` с EN keyword index;
- `.docx` с RU TOC;
- `.docx` с RU author index;
- `.docx` с EN author index;
- CSV/TXT debug artifacts для проверки каждого этапа.

## Технологии

- Python.
- Microsoft Word COM automation через `pywin32`.
- CSV/TXT debug artifacts.
- Git.
- PowerShell workflow.
- Tagged pipeline для повторяемых прогонов разных версий документа.

## Особенности

- Постраничное извлечение через Microsoft Word, а не через приблизительный парсер `.docx`.
- Раздельная обработка RU и EN блоков.
- Tagged pipeline для версий вроде `test4`, `test5`, `test6`, `test7`, чтобы артефакты разных прогонов не перезаписывали друг друга.
- Debug artifacts на каждом важном этапе: snapshot параграфов, author windows, title-author pairs, draft indexes, TOC drafts.
- Safe save и fallback save для случаев, когда итоговый Word-файл открыт или занят.
- Обработка длинных строк русского оглавления: ранний перенос заголовков, упаковка авторов по длине строки, сохранение переносов по границам авторов.
- Нормализация авторов: инициалы, порядок фамилия-инициалы, похожие кириллические/латинские символы, apostrophe cases вроде `Dement'ev`.
- Локализация EN article blocks от уже найденных RU статей, без глобального поиска по всему документу.

## Ограничения

- Работает на Windows с установленным Microsoft Word.
- Для точных страниц и финальной вставки используется Word COM, поэтому окружение должно иметь доступ к Word.
- Структура исходного документа должна быть близка к поддерживаемому шаблону.
- Документы с сильно иной разметкой требуют проверки и, возможно, настройки эвристик.
- Финальная визуальная проверка в Word всё равно нужна: переносы, отступы, номера страниц и общий вид итогового документа зависят от реального Word layout.

## Моя роль

- Проектирование pipeline для последовательной обработки большого Word-документа.
- Диагностика структуры Word-документа через snapshot и debug artifacts.
- Разработка эвристик извлечения RU/EN заголовков, авторов, ключевых слов и страниц.
- Автоматизация сборки итогового `.docx` с несколькими индексами и оглавлением.
- Контроль качества через промежуточные CSV/TXT артефакты и проверяемые summaries.
- Точечная доработка edge cases: переносы TOC, safe save, tagged artifacts, нормализация initials/homoglyph/apostrophe cases.
